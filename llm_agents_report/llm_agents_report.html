概述：LLM 与 AI 代理的演进与应用

本简报旨在深入探讨大语言模型（LLM）和人工智能（AI）代理的核心概念、关键技术、最佳实践以及在实际应用中的广泛潜力。从基础的提示工程技巧到复杂的代理架构，我们将梳理这些技术如何赋能企业和个人，提升效率，并推动创新。

提示工程：引导 LLM 的艺术

提示工程是优化 LLM 输出效果的关键。它不仅仅是编写指令，更是一种结构化和精炼输入以最大化模型性能的方法。

基础提示类型

零样本提示

不提供任何示例，直接给出任务指令

少样本提示

通过提供少量示例来演示所需的输入/输出格式

系统提示

定义模型的根本能力和首要目的

角色提示

将特定的角色或身份分配给语言模型

上下文提示

提供与当前对话或任务相关的具体细节或背景信息

进阶提示策略

步进提示

鼓励模型在回答前退一步思考更高层次的概念或原则

思维链 (CoT)

通过"让我们一步一步思考"等指令，引导模型逐步推理

自洽性

通过多次生成答案并选择最一致的结果来提高推理任务的准确性

思维树 (ToT)

一种更复杂的推理方法，涉及探索多个推理路径

ReAct

结合了推理和行动，使语言模型能够以动态的方式执行任务

自动化提示工程

利用 LLM 本身生成和评估多个提示变体，以找到最佳性能的提示

控制 LLM 输出行为的参数

温度 (Temperature)

控制令牌选择的随机程度

Top-P 采样

选择累积概率不超过特定值的顶部令牌

Top-K 采样

选择概率最高的 K 个令牌作为候选

推荐起始参数：
相对连贯且有创意： 温度 0.2, Top-P 0.95, Top-K 30
特别有创意： 温度 0.9, Top-P 0.99, Top-K 40
较少创意： 温度 0.1, Top-P 0.9, Top-K 20
单一正确答案： 温度 0

提示工程最佳实践

试验

找到合适的提示需要反复试验

使用动词

明确指出 LLM 应该执行的动作，如"Act"、"Analyze"、"Create"等

明确输出

明确指定所需的输出格式和内容

控制长度

通过配置或提示明确要求来限制响应长度

使用变量

使提示更具动态性，如使用 {city} 等占位符

试验格式

不同的格式和措辞会产生不同的结果

AI 代理：从 LLM 到自主行动

AI 代理代表着从传统独立语言模型到动态问题解决和交互的飞跃。代理是"一种应用程序，旨在通过感知其环境并利用其可用的工具进行战略性行动来达到特定目标。"

代理的核心能力

感知环境

能够理解和分析周围环境的信息

推理决策

基于逻辑和外部信息进行战略性思考

工具使用

利用可用工具执行具体行动

代理的评估方法

轨迹评估

衡量代理在实现目标过程中的每一步表现，包括完整性、准确性、效率等指标

最终响应评估

关注代理是否达到了其目标，可通过自定义成功标准来衡量

自动评分器

一个充当评判者的 LLM，根据用户定义的标准评估生成的响应，模仿人工评估

代理架构与模式

ReAct 框架

融合了推理和行动的框架，使代理能够动态执行任务

代理式 RAG

引入自主检索代理，通过迭代推理积极优化搜索，包括上下文感知查询扩展、多步骤推理和自适应源选择

重构代理

专门用于根据用户偏好和上下文调整响应的语气、风格和呈现方式，如将技术信息转换为会话性语言

护栏机制

确保代理行为符合预期和安全，包括相关性分类器、安全分类器和基于规则的保护

实际应用与案例：AI 如何赋能各行各业

LLM 和 AI 代理在各种企业和组织中展现出巨大的潜力，从内容创作到数据分析再到客户服务。

内容与营销

  • Wild Hearts Idaho：使用 Gemini in Docs 撰写社交媒体文案推广项目
  • 品牌经理利用 Gemini Advanced 生成创意标志、标语和潜在名称
  • 创建电子邮件营销序列，生成广告文案变体
  • 通信经理草拟内部沟通消息，如公司内网重新设计通知

数据管理与分析

  • Air Force Research Laboratory：利用 Google Cloud 加速科学发现，包括发现隐藏的小行星
  • Jacobsen Salt Co.：使用 Gemini in Sheets 组织不断变化的环境数据
  • Mercado Libre：使用 Vector Search 和 Vertex AI 提供精确的购物搜索结果
  • Tchibo：利用 Vertex AI 和 BigQuery 优化需求预测，每天生成超过 600 万个预测
  • Spoon Guru：使用 Vertex AI 处理每天高达 140 亿个数据点，帮助确定产品适配性

客户服务与内部运营

  • BBVA：员工创建了 2900+ 自定义 GPT，显著缩短项目和流程时间
  • 信贷风险团队更快更准确地确定信用度，法律团队每年回答 4 万个政策问题
  • NotCo：开发了由 Gemini 驱动的对话式 AI 聊天机器人，查询 SAP 系统
  • Persistent Systems：开发 FinSight 工具，利用 Gemini 模型查询财务文档

代码生成与开发辅助

  • 自动代码生成和注释添加
  • 代码解释和功能说明
  • 跨语言代码翻译
  • 代码调试和审查优化
重要提醒

由于 LLM 无法推理并重复训练数据，因此在运行和测试代码之前阅读和测试代码至关重要。

多模态提示

多种输入格式的融合

多模态提示是一种"使用多种输入格式来指导大型语言模型"的技术，而不仅仅依赖于文本。这可以包括"文本、图像、音频、代码甚至其他格式的组合"，具体取决于模型的能力和手头的任务。

总结

本简报全面回顾了提示工程的各种技术，从基本的零样本、少样本到系统、角色和上下文提示，再到更复杂的步进提示、思维链、自洽性、思维树、ReAct 和自动化提示工程。同时，深入探讨了 LLM 输出控制参数如温度、Top-K 和 Top-P 的作用及其相互影响，并提供了实用的最佳实践指导。

此外，简报强调了 AI 代理作为下一代 AI 应用的重要性，它们能够自主感知、推理和行动，并通过 ReAct、Agentic RAG 和重构代理等架构实现更强大的功能。我们还探讨了代理评估的重要方法，包括轨迹评估、最终响应评估和自动评分器。

在实际应用层面，我们看到了众多知名企业如 BBVA、NotCo、Mercado Libre、Tchibo 等如何在金融、零售、制造、行政、人力资源、营销、项目管理、销售等多个领域成功应用 LLM 和 AI 代理，通过自动化、优化决策和提升效率，解决了实际业务问题。

最终，无论是单一的 LLM 应用还是复杂的 AI 代理系统，有效的提示工程和对模型行为的精准控制都是实现预期结果的关键。未来的发展将继续探索多模态交互和更智能的代理协作，以解锁更广泛的 AI 能力。
